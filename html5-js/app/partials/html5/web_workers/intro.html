• Web Workers allow to execute tasks in background threads
  • To create a Web Worker, you just need to tell the script it has to execute
  • A worker has its own execution environment:
    • It doesn't share its data with other workers
    • It doesn't have access to the dom
    • It can communicate with its parent: again, data is copied, not shared
  • You could see a worker a little bit like a batch and imagine a lot of use cases
    • Continuously do AJAX calls to get recent data
    • Clean your indexedDB by removing unused data
    • Make intensive calculation without blocking the user...
