When using cache manifest, at least two refresh are necessary to get new assets from server

  • Why ?
    • It's a safety mechanism which ensure that all assets are successfully downloaded before transfer it to the active cache
      • At first request, the assets are stored in a temporary Store
    • If there is a problem with one asset, the cache is not updated and you still get your old working application
    • Note : the front page is automatically cached. So user will have to refresh twice to get new contents !
      • You can check that in the developer tools of your Browser
