The IndexedDB API is an event driven API. Events are used at different levels, and you can interact with them by using custom handlers. Most used events are success and error, but some others can also be used depending on the operation you are doing. Below are some examples.

  • success and error: can be handled when you open the database, read object stores or create object stores and more generally each time a request is used
  • upgradeneeded : you try to open a database which doesn't exist or with a version id higher than its current version id
  • blocked : you try to open a DB which is already opened
  • abort,complete,error: Can be handled at the transaction level to know its state
