• Give the ability to find DOM elements using CSS selectors, which are more powerfull
  • Similar to the jQuery Selector API
  • Advatage compared to jquery
    • Faster because directly implemented at the navigator level
    • Evident but... you don't need jQuery to use it
  • Disadvatage compared to jquery
    • Depnds on the implementations of CSS selectors on your browser
    • For example, ie8 will not use CSS3 selectors, but CSS2.1 and so pseudo classes will not be available
  • Only 2 methods, querySelector and querySelectorAll, for single or multiple results. querySelector will not parse all the DOM...
partial interface Document { Element? querySelector(DOMString selectors); NodeList querySelectorAll(DOMString selectors); } partial interface DocumentFragment { Element? querySelector(DOMString selectors); NodeList querySelectorAll(DOMString selectors); } partial interface Element { Element? querySelector(DOMString selectors); NodeList querySelectorAll(DOMString selectors); }
