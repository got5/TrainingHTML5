2 versions of the API

  • The asynchronous one, for user actions: We deal with callbacks (lot of them)
  • The synchronous one, makes the development easier, but could block the user, so to use with WebWorkers only

Some remarks

  • The data are sandboxed: Access limited to the same origin domain (like others html5 storage api)
  • Capacity
    • No recommendation on allocated size from W3C specification
    • It depends of browser but you can store pretty large file (over 50Mb/file on Chrome)
